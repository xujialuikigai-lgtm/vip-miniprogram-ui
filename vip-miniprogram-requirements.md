# 会员权益中心小程序 - 需求文档

**版本**：v2.0  
**最后更新**：2026年6月16日  
**基于**：`vip-miniprogram-ui.html` 原型（含自动开通链路）

---

## 一、产品概述

### 1.1 产品定位

虚拟会员代充小程序，为用户提供视频、音乐、网盘、工具类会员的官方直充服务。通过第三方接口实现自动开通，支付后系统自动提交，减少人工干预。

### 1.2 核心卖点

- **自动开通**：支付成功后系统自动调用第三方接口开通，无需人工处理
- **快速到账**：常规 1-5 分钟到账
- **失败退款**：接口开通失败按原路退回
- **不自动续费**：支付一次后不再扣款
- **订单可查**：全流程透明追踪，含接口处理状态

### 1.3 设计基调

正规、清爽、可信赖。强调信任建立与流程透明，弱化营销感。

### 1.4 用户角色

| 角色 | 说明 | 权限 |
|------|------|------|
| 普通用户 | 微信用户 | 浏览商品、下单、查看自己的订单 |
| 管理员 | 白名单微信号 | 处理订单、管理商品、查看数据、重试接口 |

> 客服通过企业微信二维码方式提供服务，入口在"我的"页面导航栏。

## 三、用户端需求

### U1 首页

#### 模块组成

**① 顶部导航栏**
- 左侧：无（标题居中）
- 中间：标题"会员权益中心"
- 右侧：无

**② 搜索栏**
- 占位文字："搜索视频、音乐、网盘会员"
- 点击进入搜索结果页（U8）

**③ 信任 Hero 区**
- 暗色卡片，主标题"会员权益充值中心"
- 副标题："支付成功后系统自动提交开通，订单全程可查，失败原路退款。"
- 三个数据指标：
  - 1-5 分钟（常规到账）
  - 8.2万+（累计订单，需后台可配置）
  - 0 续费（不会扣款）

**④ 实时购买播报**
- 滚动展示最近购买记录
- 文案格式：`138****6066 刚刚购买 爱奇艺黄金 VIP 30天`
- 数据来源：真实订单的脱敏展示，3-5 秒切换一条

**⑤ 分类 Tab**
- 选项：热门 / 视频 / 音乐 / 网盘 / 工具
- 默认选中"热门"，切换 Tab 刷新下方商品列表
- 支持横向滑动，超出屏幕的 Tab 可滑动查看

**⑥ 商品列表**

每个商品卡片包含：
- 品牌图标块（左侧 66×66，按分类配色：视频绿、音乐蓝紫、网盘橙）
- 商品名（如"爱奇艺黄金 VIP 30天"）
- 标签（绿色小标签）：权益直充 / 1-5分钟到账 / 失败退款 / 自动开通 / 热门推荐 等
- 销量与说明（如"已售 17600+｜不自动续费"）
- 价格区：现价（金色大字）+ 划线原价
- 查看详情按钮（金色渐变）

商品列表为竖向排列，无限下拉加载（首期可一次性加载所有上架商品）。

**⑦ 底部导航**
- 4 个 Tab：首页 / 分类 / 订单 / 我的
- 当前页高亮（首页时高亮首页）

#### 页面跳转
- 点击商品卡片任意区域 → 商品详情页（U5），传商品 ID
- 点击"查看详情"按钮 → 商品详情页（U5），传商品 ID

### U3 我的订单

#### 模块组成

**① 顶部导航栏**
- 返回按钮
- 标题"我的订单"

**② 状态筛选 Tab**
- 全部 / 开通中 / 开通成功 / 退款
- 默认选中"全部"

**③ 订单卡片列表**

每张订单卡片包含：
- 商品名称
- 订单状态（不同状态不同颜色：开通成功-绿色、开通中-绿色、开通失败-红色）
- 充值账号（脱敏，如"138****6066"）
- 订单号
- 支付金额 + 下单时间（或预计到账时间）
- 操作按钮（按状态变化）：

| 订单状态 | 显示按钮 |
|---------|---------|
| 开通成功 | 查看详情 / 再买一次 |
| 开通中 | 查看进度 / 刷新进度 |
| 开通失败 | 查看原因 / 重新购买 |
| 已退款 | 查看详情 / 再买一次 |

**④ 订单说明**

底部固定提示：
1. 开通中订单请等待系统返回结果，重复购买不会加速当前订单
2. 失败订单会按原支付方式退款，具体到账以微信支付为准

**⑤ 底部导航**
- 当前高亮"订单"

#### 页面跳转
- 点击"查看详情"或卡片本身 → 订单详情页（用户视角）
- 点击"再买一次" → 商品详情页（U5），自动选中相同套餐
- 点击"重新购买" → 商品详情页（U5），保留套餐但清空手机号
- 点击"刷新进度" → 重新查询订单状态，Toast 提示"已刷新最新开通进度"

### U4 我的（个人中心）

#### 模块组成

**① 顶部导航栏**
- 左侧：无（标题居中）
- 标题"我的"
- 右侧：客服图标（点击弹出企业微信二维码弹窗）

**② 用户信息区**
- 圆形头像（默认显示"U"或微信头像）
- 用户名（默认"微信用户"，已授权后显示微信昵称）
- 副文案："已授权登录｜订单和售后进度可追踪"

**③ 快速统计卡片（3 列）**
- 全部订单：显示用户全部订单数
- 处理中：显示开通中订单数
- 待退款：显示退款中订单数

点击任一卡片 → 跳转我的订单（U3）并自动切换到对应 Tab。

**④ 常用功能菜单**

每行包含菜单名称 + 副文案：
- 我的订单 → 查看全部 →
- 购买须知 → 退款/到账规则 →
- 账号填写说明 → 避免充错 →
- 平台公告 → 维护与优惠 →

点击进入对应内容页（购买须知、账号填写说明、平台公告为静态富文本页）。

**⑤ 管理员入口**

> 仅白名单用户可见，普通用户此模块不显示。

包含 3 张卡片（暗色背景，2 列布局）：
- **数据看板**：销售额、单量、开通成功率和接口失败
- **订单处理**：接口状态、重试开通、失败原因和退款
- **商品运营**：接口商品、成本售价、上下架和分类管理

**⑥ 服务保障**

3 条说明：
1. 会员商品不会自动续费，支付后系统按所选周期自动提交开通
2. 订单问题可通过右上角客服图标联系企业微信客服
3. 管理员入口仅白名单可见，普通用户不会显示

**⑦ 底部导航**
- 当前高亮"我的"

### U5 商品详情页

#### 模块组成

**① 顶部导航栏**
- 返回按钮
- 标题为商品品牌名（如"爱奇艺视频"）

**② 商品 Hero 区**
- 暗色卡片
- 左侧大号品牌图标
- 右侧：品牌名（如"爱奇艺 & 奇异果 TV"）+ 副文案"支付后自动提交开通｜规则透明｜失败退款｜不会自动续费" + 标签（到账快、订单可追踪）

**③ 会员类型选择**
- 标题"会员类型"
- 3 列横向卡片：黄金 VIP / 白金 VIP / 星钻 VIP
- 选中态：金色描边 + 浅色背景
- 切换会员类型时，下方套餐价格刷新

> 不是所有商品都有 3 个类型，部分商品只有 1-2 个类型，需后台配置。

**④ 套餐选择**
- 标题"选择套餐"，右侧默认提示"默认 30天"
- 2 列网格，常见套餐：7天 / 30天 / 3个月 / 1年
- 每个套餐卡片：左上角折扣标签（如"7折"）+ 中间套餐周期 + 下方价格
- 默认选中 30 天，选中态金色描边
- 切换套餐时，底部支付栏的合计金额同步更新

**⑤ 关键须知**

列表展示（条数可后台配置）：
1. 支付成功后系统自动提交开通，大部分订单 1-5 分钟到账
2. 账号填错或商品买错不支持无理由退款；开通失败会原路退款

**⑥ 权益对比表**

表格展示三种会员类型的差异（数据来自商品配置）：

| 权益 | 黄金 | 白金 | 星钻 |
|------|------|------|------|
| 电视端 | 不支持 | 支持 | 支持 |
| 播放 | 1台 | 2台 | 3台 |
| 登录 | 5台 | 8台 | 10台 |

**⑦ 底部支付栏（固定）**
- 左侧：手机号输入框（占位文字"请输入绑定手机号"，已输入时显示号码）
- 右侧：购买按钮（金色渐变）

> 注意：新 UI 将充值账号输入从独立区域移至底部固定栏，与购买按钮同行展示。

#### 业务规则
- 套餐价格变更后，已停留在该页面的用户提交订单时需校验最新价格，价格变化时提示用户重新选择
- 手机号未填写时，"购买"按钮置灰不可点击
- 手机号输入校验：11 位大陆手机号格式

#### 页面跳转
- 点击"购买" → 唤起购买核对弹窗（U6）

### U9 空状态与异常状态（新增）

> 以下为各页面的空状态和异常状态设计，非独立页面。

#### 订单为空
- 圆形图标 + "还没有订单"
- 副文案："购买会员后，充值进度和退款状态会显示在这里。"

#### 分类无商品
- 圆形图标 + "当前分类暂无商品"
- 副文案："管理员上架商品后，这里会自动展示。"

#### 加载状态
- 骨架屏：品牌图标占位 + 文字行占位
- 使用渐变动画效果

#### 异常状态
- **网络断开**：显示重试按钮，保留用户已输入手机号，不清空表单
- **手机号格式错误**：输入框下方红字提示"请输入 11 位大陆手机号"

#### 用户侧操作反馈
- **刷新进度**：点击后重新查询订单状态，Toast 提示"已刷新最新开通进度"
- **再买一次（商品已下架）**：弹窗提示"该商品暂不可购买，可返回分类选择其他商品"

### U10 订单详情页（用户视角）

#### 模块组成

**① 顶部导航栏**
- 返回按钮
- 标题"订单详情"
- 右侧订单入口

**② 状态 Hero 区**
- 暗色卡片
- 主标题：当前订单状态（如"开通中"）
- 描述文案（如"支付成功，系统已为 178****0822 自动提交腾讯视频 VIP 3个月开通。"）
- 三个指标：实付金额 / 下单时间 / 预计到账

**③ 订单进度（时间轴）**
- 支付成功 + 时间 + 描述
- 提交开通 + 描述（云函数已调用第三方开通接口）
- 接口处理中 + 描述（正在等待平台返回开通结果）
- 等待完成（灰色，未完成节点）

**④ 订单信息卡片**
- 订单号（可复制）
- 商品名
- 充值账号（脱敏）
- 当前状态（带颜色）
- 操作按钮：复制订单号 / 刷新进度

**⑤ 异常说明**

## 五、关键业务规则

### 5.1 订单状态机（自动开通版）

```
待支付 ──支付成功──→ 已支付
   │                    │
   │              云函数调用接口
   │                    ↓
   │               开通中（接口处理中）
   │                    │
   │         ┌──────────┼──────────┐
   │         ↓                     ↓
   │    开通成功               接口失败
   │   （通知用户）          （记录返回码）
   │                               │
   │                          发起退款
   │                               ↓
   │                           退款中
   │                               ↓
   │                           已退款
   │
   └──超时/取消──→ 已取消
```

### 5.2 自动开通链路

1. 用户支付成功 → 微信支付回调触发云函数
2. 云函数锁定价格、创建订单记录
3. 云函数携带密钥调用第三方开通接口，传入接口商品 ID / 接口 SKU + 用户手机号
4. 记录第三方单号（三方单号），订单进入"开通中"
5. 轮询或接收第三方回调：
   - 成功 → 标记"开通成功"，通知用户
   - 失败 → 记录返回码和失败原因，进入退款流程
6. 超时（10 分钟）未返回 → 系统自动重新查询接口结果

### 5.3 价格与成本规则

- 用户下单时，订单实付金额即被锁定，后续套餐改价不影响该订单
- 每个套餐配置售价和成本价，系统自动计算利润
- 用户停留在商品详情页时间过长时，提交订单前需校验当前价格是否一致
- 管理端展示：售价、接口成本、单单利润

### 5.4 隐私与脱敏

- 用户端展示的手机号一律脱敏（`138****6066`）
- 管理端订单列表脱敏展示，订单详情页可看到完整手机号
- 订单号、三方单号、充值账号支持长按复制

### 5.5 通知规则

- 订单状态变更时通过订阅消息通知用户：
  - 支付成功（系统已提交开通）
  - 开通成功
  - 开通失败（含退款进度）
  - 退款到账

### 5.6 管理员权限

- 管理员入口在"我的"页面仅对白名单用户显示
- 普通用户即使知道页面路径，访问管理端页面时也会校验权限并阻止
- 所有管理员操作和系统自动操作均需记录到操作日志

### 5.7 防误操作

- 标记订单成功 / 发起退款前需弹窗二次确认
- 退款操作必须选择失败原因分类 + 可选补充说明
- 商品上下架切换前需确认弹窗
- 改套餐价保存前弹窗确认："新价格只影响后续订单；已支付订单仍按支付时锁定价格处理"
- 商品编辑保存前对必填项做校验

### 5.8 接口安全

- 第三方开通接口密钥仅保存在云函数/云托管服务端
- 小程序前端不存储、不展示任何密钥信息
- 所有接口调用记录保存到接口日志，包含请求时间、返回码、返回描述、耗时
